# ClaudeBar 디자인 스펙

날짜: 2026-07-21
상태: 사용자 승인됨 (구현 계획 작성 전)

## 개요

Claude Code 사용량을 귀여운 픽셀아트 캐릭터와 함께 macOS 메뉴바에 표시하는 오픈소스 앱.

- 프로젝트명: **ClaudeBar**
- 저장소: `~/Workspace/claude-bar` (GitHub 공개 예정)
- 라이선스: MIT
- README: 영어 기본 + 한국어 병기
- 대상 사용자: Claude Code Pro/Max 플랜 사용자 (제작자 본인 + 오픈소스 배포)

## 요구사항

1. 메뉴바 상시 표시: 픽셀아트 캐릭터 + 한도 잔여량 %
2. 캐릭터 표정이 잔여량에 따라 5단계로 변화
3. Claude Code 작업 중일 때 캐릭터 애니메이션
4. 클릭 시 팝오버: 플랜 한도(세션/주간), 토큰 통계, 활동 상태 전부 표시
5. 한도 임박 알림 (20%, 5%) + 한도 소진 시 재밌는 "퇴근" 알림
6. 로그인 시 자동 시작
7. Homebrew로 설치 가능한 오픈소스 배포

## 아키텍처

Swift Package Manager 기반 executable. Xcode 프로젝트 파일 없음 — `swift build` 한 줄로 빌드되어 오픈소스 기여 장벽이 낮다. SwiftUI + AppKit, `LSUIElement`로 Dock 아이콘 없는 메뉴바 전용 앱.

### 컴포넌트

| 컴포넌트 | 책임 | 의존성 |
|---|---|---|
| `UsageFetcher` | 키체인의 Claude Code OAuth 토큰으로 사용량 API 호출 → 세션/주간 한도 %와 리셋 시각 | Security, URLSession |
| `TranscriptScanner` | `~/.claude/projects/**/*.jsonl` 파싱 → 오늘/이번 주 토큰·API 환산 비용, FSEvents로 작업 중 감지 | Foundation, FSEvents |
| `CharacterRenderer` | 코드 내 비트맵 배열로 픽셀아트 NSImage 생성 (표정 5단계 + 애니메이션 프레임 + 물음표) | AppKit |
| `StatusItemController` | NSStatusItem 아이콘·텍스트 갱신, 애니메이션 타이머 | AppKit |
| `PopoverView` | 클릭 시 SwiftUI 상세 화면 | SwiftUI |
| `NotificationManager` | 임계값 알림 + 퇴근 알림 (UNUserNotifications) | UserNotifications |
| `LaunchAtLogin` | SMAppService 기반 자동 시작 토글 | ServiceManagement |

각 컴포넌트는 프로토콜 뒤에 두어 단위 테스트 시 목 주입 가능하게 한다.

## 캐릭터

- Clawd(Claude Code 마스코트) 풍의 주황색 픽셀아트 생물. 18×18pt 메뉴바 아이콘 (@2x 36px 렌더).
- 픽셀 데이터는 코드 내 2차원 배열로 정의 — 이미지 에셋 파일 불필요.
- 다크/라이트 메뉴바 모두에서 보이도록 외곽선 처리. 몸통 주황색은 유지 (템플릿 이미지 아님).

### 표정 단계 (잔여량 기준)

| 잔여량 | 표정 |
|---|---|
| 60% 이상 | 평온 |
| 40–60% | 미소 |
| 20–40% | 걱정 |
| 5–20% | 힘듦 |
| 5% 미만 | 기절 |
| 데이터 없음 | 물음표 표정 + `--%` |

### 애니메이션

- 작업 중(활성 세션 감지) 시 2–3프레임 꼼질꼼질 애니메이션, 0.5초 간격.
- 유휴 시 정지 (배터리 배려). 타이머는 작업 중일 때만 돌린다.

## 메뉴바 표시

`[캐릭터] 72%` 형식. 표시 %는 **세션 잔여량과 주간 잔여량 중 더 낮은 쪽** (실제로 먼저 막히는 한도).

## 팝오버 레이아웃 (위→아래)

1. 큰 캐릭터 + 상태 한 줄 (예: "여유로워요~")
2. 5시간 세션 한도 게이지 바 + 리셋까지 남은 시간
3. 주간 한도 게이지 바 + 리셋 요일/시각
4. 토큰 통계: 오늘/이번 주 토큰 수, API 환산 비용
5. 현재 활동 중인 프로젝트 (활성 세션이 있을 때)
6. 설정: 알림 임계값(20%/5%) 토글, 로그인 시 자동 시작 토글, 수동 새로고침, 종료

## 데이터 소스와 갱신

### 플랜 한도 (주 데이터)

- Claude Code가 macOS 키체인에 저장한 OAuth 자격증명을 읽어 `/usage` 명령이 사용하는 사용량 API 호출.
- 갱신 주기: 유휴 시 60초, 작업 중 30초.
- 최초 실행 시 키체인 접근 허용 팝업 1회 발생 — README에 명시.
- **리스크와 폴백**: 비공개 엔드포인트이므로 구현 시점에 실제 요청/응답을 확인해 맞춘다. 사용 불가 시 세션 시작 시각 기반 추정(ccusage blocks 방식)으로 폴백.

### 토큰 통계 (로컬)

- `~/.claude/projects/**/*.jsonl`의 usage 필드 집계. 키체인/API와 무관하게 항상 동작.
- FSEvents로 디렉터리 감시, 변경 시 디바운스(수 초) 후 증분 재집계.

### 작업 중 감지

- 최근 N초(기본 30초) 내에 수정된 세션 jsonl 파일이 있으면 "작업 중". 해당 파일의 프로젝트 경로를 활동 중 프로젝트로 표시.

## 알림

- 잔여량이 임계값(20%, 5%) 아래로 **처음 내려갈 때 1회만** macOS 알림. 한도 리셋 시 재활성화.
- **퇴근 알림**: 잔여량 0%(한도 소진) 도달 시 재밌는 메시지 랜덤 발송 — 예: "오늘 몫은 다 썼어요! 이제 퇴근하세요 🦞", "Clawd도 쉬고 싶대요. 내일 봐요!", "한도 끝! 산책 어때요?" 캐릭터는 기절 표정.
- 알림 권한은 최초 실행 시 요청.

## 에러 처리

| 상황 | 동작 |
|---|---|
| 키체인 접근 거부 / Claude Code 미로그인 | 메뉴바 `--%` + 물음표 표정, 팝오버에 안내 문구. 로컬 토큰 통계는 계속 표시 |
| 사용량 API 실패 (네트워크 등) | 마지막 값 유지 + 팝오버에 "마지막 갱신 시각" 표시, 다음 주기에 재시도 |
| jsonl 파싱 실패 (포맷 변경 등) | 해당 라인/파일 건너뛰고 나머지 집계, 크래시 금지 |

## 배포 (Homebrew)

- GitHub 공개 저장소 + GitHub Actions로 태그 릴리즈 시 자동 빌드.
- 개인 tap: `brew tap bmpark/claude-bar && brew install claude-bar`. Formula가 소스에서 빌드하므로 Apple 개발자 계정·공증 없이 Gatekeeper 경고 없음.
- 미리 빌드된 cask 배포는 공증 필요로 1단계 제외. homebrew-core 등록은 인지도 확보 후 도전.

## 테스트

- 단위 테스트(`swift test`): jsonl 파서(픽스처 파일 기반), 한도 % 계산, 표정 단계 매핑, 알림 임계값 상태 머신(1회 발송/리셋 재활성화).
- UI(메뉴바·팝오버·애니메이션)는 수동 확인.

## 범위 제외 (YAGNI)

- Windows/Linux 지원
- 여러 Claude 계정 전환
- 사용량 히스토리 차트/그래프
- 메뉴바 외 위젯(알림 센터 위젯 등)
